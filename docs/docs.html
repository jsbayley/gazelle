Installation

Prerequisites

  • .NET 9 SDK or later
  • F# support (included with .NET SDK)

Install from NuGet

Add Gazelle to your project 
dotnet add package Gazelle --version 0.0.4

Build from Source

Clone and build 
git clone https://github.com/jsbayley/gazelle.git
cd gazelle
dotnet build

# Install CLI tool globally (works on all platforms)
dotnet tool install --global --add-source ./cli/bin/Release Gazelle.CLI

Cross-Platform Support: Gazelle works on Windows, macOS, and Linux. ETABS integration is available on Windows only, with graceful error handling on other platforms.

Model Creation

Basic Syntax

Create models with examples 
gz create [output-file] --example [type] [options]

Available Examples

Truss Structure

3-node triangular truss 
gz create truss.json --example truss --span 10.0 --height 4.0 --loads 25.0

Creates a truss with 10.0m span, 4.0m height, and 25.0 kN downward load at apex.

Cantilever Beam

2-node cantilever beam 
gz create beam.json --example cantilever --span 6.0 --loads 50.0

Creates a 6.0m cantilever with fixed support and 50.0 kN tip load.

Portal Frame

4-node portal frame 
gz create frame.json --example portal --span 12.0 --height 4.0 --loads 15.0,100.0

Creates a 12.0m x 4.0m portal frame with wind (15.0 kN) and dead loads (100.0 kN).

Model Analysis

Static Analysis

Linear static analysis 
gz analyze model.json --type static --output results.json

Performs linear static analysis calculating:

  • Node displacements
  • Support reactions
  • Strain energy
  • Maximum displacement and reaction values

Modal Analysis

Eigenvalue analysis 
gz analyze model.json --type modal

Performs eigenvalue analysis determining natural frequencies and mode shapes.

Analysis Options

  • --type: Analysis type (static, modal, dynamic)
  • --output, -o: Output file for results
  • --solver: Solver type (auto, cholesky, lu)
  • --tolerance: Convergence tolerance (default: 1e-9)
  • --max-iterations: Maximum solver iterations (default: 1000)

Model Validation

Comprehensive model checking 
gz validate model.json

Performs comprehensive model checking:

  • Structural integrity verification
  • Node connectivity analysis
  • Constraint sufficiency checking
  • Load case validation
  • Material property verification
Model information display 
gz info model.json

Gazelle.IO Library

Gazelle.IO is a companion library that provides robust IO operations and ETABS interop for the Gazelle structural analysis engine.

Quick Start

# Build the library
dotnet build io/Gazelle.IO.fsproj

# Use in your projects
dotnet add reference io/Gazelle.IO.fsproj

ETABS Integration

Connect to ETABS for advanced structural modeling and analysis:

open Gazelle.IO.ETABS

// Launch ETABS and create a new model
let result = ETABS.start()

match result with
| Ok (etabsApp, sapModel) ->
    printfn "‚úÖ ETABS connected successfully"
    
    // Initialize a clean model environment
    ETABS.initialise sapModel
    
    // Refresh the ETABS view
    ETABS.refreshView sapModel
    
    // Close ETABS (with save)
    ETABS.close result SaveFile
    
| Error e ->
    printfn "‚ùå Failed to connect to ETABS: %A" e

Error Handling Pattern

All operations use the robust Result<'T, IOError> pattern:

open Gazelle.IO

// Interactive user input with validation
let userChoice = IO.askForUserInput "Enter your choice:" validateInput

match userChoice with
| Ok validChoice -> 
    printfn "Processing: %A" validChoice
| Error err ->
    IOError.print err

Worked Examples

Example 1: Simple Truss Analysis

Create and analyze an 8m span truss with 30 kN load:

Complete truss workflow 
# Create model
gz create example1.json --example truss --span 8.0 --height 3.0 --loads 30.0

# Validate model  
gz validate example1.json

# Run static analysis
gz analyze example1.json --type static --output example1_results.json

# View results
gz info example1.json

Expected Results:

  • Maximum displacement: ~2.5e-5 m
  • Support reactions: ~15 kN each
  • Analysis time: <5 microseconds

Example 2: Cantilever Beam Design

Design a 5m cantilever beam for 40 kN tip load:

Cantilever analysis 
# Create beam model
gz create cantilever.json --example cantilever --span 5.0 --loads 40.0

# Analyze deflection
gz analyze cantilever.json --output cantilever_results.json

# Check natural frequencies
gz analyze cantilever.json --type modal

Expected Results:

  • Tip deflection: ~6.0e-5 m
  • Fixed support reaction: 40 kN
  • First natural frequency: ~10 Hz

Example 3: Portal Frame Analysis

Analyze a 10m x 4m portal frame with combined loading:

Portal frame with wind and dead loads 
# Create frame with wind and dead loads
gz create portal.json --example portal --span 10.0 --height 4.0 --loads 20.0,80.0

# Perform comprehensive analysis
gz validate portal.json
gz analyze portal.json --type static --output portal_static.json
gz analyze portal.json --type modal

# Review model properties
gz info portal.json

Expected Results:

  • Maximum displacement: ~9.0e-5 m
  • Base reactions: Variable based on load distribution
  • Multiple natural frequencies: 10-25 Hz range

Units and Conventions

Standard Units

  • Length: meters (m)
  • Force: Newtons (N)
  • Stress: Pascals (Pa)
  • Mass: kilograms (kg)
  • Time: seconds (s)
  • Frequency: Hertz (Hz)
  • Load input: kilonewtons (kN) - automatically converted to N
